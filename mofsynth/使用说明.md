# MOFSynth 可合成性预测工具 - 使用说明

## 📦 包内容

本工具包包含预测MOF可合成性所需的所有文件：

```
mofsynth_predict_package/
├── predict_PU_learning.py          # 主预测脚本
├── generate_crystal_graph.py       # 晶体图生成脚本
├── atom_init.json                  # 原子特征初始化文件 (92维)
├── trained_models/                 # 100个预训练模型 (170MB)
│   ├── checkpoint_bag_1.pth.tar
│   ├── checkpoint_bag_2.pth.tar
│   └── ... (共100个模型)
├── cgcnn/                          # CGCNN模型架构代码
│   ├── model_PU_learning.py
│   └── data_PU_learning.py
├── example_cif_files/              # 示例目录
│   └── id_prop.csv                 # 示例ID属性文件
└── 使用说明.md                     # 本文件
```

## 🚀 快速开始

### 步骤1: 准备您的MOF结构文件

创建一个工作目录，例如 `my_mofs/`：

```bash
mkdir my_mofs
cd my_mofs
```

#### 1.1 放置CIF文件

将您的MOF CIF文件复制到此目录：
```bash
cp /path/to/your/mof_1.cif my_mofs/
cp /path/to/your/mof_2.cif my_mofs/
# ... 复制所有CIF文件
```

#### 1.2 创建id_prop.csv文件

创建一个名为 `id_prop.csv` 的文件，格式如下：

```csv
crystal_id,synthesizable
mof_1,0
mof_2,0
mof_3,0
```

**重要说明：**
- 第一列 `crystal_id`：必须与CIF文件名匹配（不含`.cif`扩展名）
- 第二列 `synthesizable`：**全部设为0**（表示待预测）
- **不要**包含表头（某些脚本要求），但包含表头也可以

#### 1.3 复制atom_init.json

```bash
cp /path/to/mofsynth_predict_package/atom_init.json my_mofs/
```

### 步骤2: 生成晶体图

使用 `generate_crystal_graph.py` 为每个CIF文件生成图表示：

```bash
python /path/to/mofsynth_predict_package/generate_crystal_graph.py \
    --cifs ./my_mofs \
    --n 12 \
    --r 8 \
    --f ./my_crystal_graph
```

**参数说明：**
- `--cifs`: 包含CIF文件和id_prop.csv的目录
- `--n`: 最大邻居数（默认12，建议保持）
- `--r`: 截断半径，单位Å（默认8，建议保持）
- `--f`: 输出目录（保存生成的图文件）

**预期输出：**
- 在 `my_crystal_graph/` 目录下生成多个`.pickle`文件
- 每个CIF对应一个pickle文件

### 步骤3: 运行预测

使用预训练的100个模型进行集成预测：

```bash
python /path/to/mofsynth_predict_package/predict_PU_learning.py \
    --bag 100 \
    --graph ./my_crystal_graph \
    --cifs ./my_mofs \
    --modeldir /path/to/mofsynth_predict_package/trained_models
```

**参数说明：**
- `--bag`: 使用的模型数量（100表示使用全部模型）
- `--graph`: 晶体图目录（步骤2生成的）
- `--cifs`: CIF文件目录
- `--modeldir`: 训练模型目录

**预期输出：**
- 在当前目录生成 `test_results_ensemble_100models.csv`
- 包含每个MOF的CLscore预测结果

### 步骤4: 查看结果

打开 `test_results_ensemble_100models.csv`：

```csv
id,CLscore,bagging
mof_1,0.7834,100
mof_2,0.5212,100
mof_3,0.2345,100
```

## 📊 结果解读

### CLscore 含义

| CLscore范围 | 可合成性 | 说明 |
|------------|---------|------|
| **> 0.7** | 高可合成性 | 类似CSD实验已合成的MOF |
| **0.5-0.7** | 中等可合成性 | 有一定合成可能性 |
| **< 0.5** | 低可合成性 | 类似hMOF假设性结构，合成困难 |

### 参考基准

根据论文测试结果：
- **CSD实验结构** (1000个): 平均CLscore = 0.7200
  - 87.9% 的结构 CLscore > 0.7
  - 0% 的结构 CLscore < 0.5

- **hMOF假设结构** (1000个): 平均CLscore = 0.4904
  - 6.9% 的结构 CLscore > 0.7
  - 52.4% 的结构 CLscore < 0.5

## 💡 完整示例

假设您有3个MOF文件：`mof_a.cif`, `mof_b.cif`, `mof_c.cif`

```bash
# 1. 创建工作目录
mkdir test_prediction
cd test_prediction

# 2. 复制CIF文件
cp /path/to/mof_*.cif ./

# 3. 创建id_prop.csv
cat > id_prop.csv << EOF
mof_a,0
mof_b,0
mof_c,0
EOF

# 4. 复制atom_init.json
cp /path/to/mofsynth_predict_package/atom_init.json ./

# 5. 生成晶体图
python /path/to/mofsynth_predict_package/generate_crystal_graph.py \
    --cifs . \
    --n 12 \
    --r 8 \
    --f ./crystal_graph

# 6. 运行预测
python /path/to/mofsynth_predict_package/predict_PU_learning.py \
    --bag 100 \
    --graph ./crystal_graph \
    --cifs . \
    --modeldir /path/to/mofsynth_predict_package/trained_models

# 7. 查看结果
cat test_results_ensemble_100models.csv
```

## ⚠️ 常见问题

### Q1: 提示找不到某个模块
```bash
# 安装缺失的模块
pip install torch pymatgen numpy pandas scikit-learn tqdm h5py joblib
```

### Q2: 提示 "atom_init.json does not exist!"
**解决方法：** 确保将 `atom_init.json` 复制到CIF文件所在目录

### Q3: 提示 "not find enough neighbors to build graph"
**解决方法：** 增加截断半径 `--r 10`

### Q4: 内存不足
**解决方法：** 减小批处理大小
```bash
python predict_PU_learning.py \
    --b 32 \  # 默认64，减小到32或16
    --bag 100 \
    --graph ./crystal_graph \
    --cifs ./my_mofs \
    --modeldir ./trained_models
```

### Q5: 预测速度慢
**解决方法：**
- 使用GPU（自动检测CUDA）
- 减少模型数量（降低精度但加快速度）：`--bag 50`

### Q6: CIF文件格式不正确
**要求：**
- CIF文件必须包含完整的晶体结构信息
- 原子坐标必须明确
- 建议使用标准CIF格式（v1.1）

## 📚 技术细节

### 模型架构
- **类型**: Crystal Graph Convolutional Neural Network (CGCNN)
- **特征维度**: 92维原子特征 (atom_init.json)
- **图卷积层**: 3层
- **集成方法**: Bootstrap Aggregating (100个模型)

### PU-Learning框架
- **正样本 (P)**: 实验已合成的MOF (CSD数据库)
- **未标记样本 (U)**: 假设性MOF (hMOF数据库)
- **训练策略**: 每个模型从U中随机采样负样本

### 预测原理
1. CIF文件 → 晶体图 (原子节点 + 键连接)
2. 晶体图 → 原子特征 (92维向量)
3. CGCNN → 图级别特征
4. 100个模型投票 → 最终CLscore (0-1)
